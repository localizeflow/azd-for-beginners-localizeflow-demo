## Επισκόπηση Μαθήματος

Κατακτήστε το Azure Developer CLI (azd) μέσα από δομημένα κεφάλαια σχεδιασμένα για προοδευτική μάθηση. **Ειδική έμφαση στην ανάπτυξη εφαρμογών AI με ενσωμάτωση Microsoft Foundry.**

### Γιατί αυτό το μάθημα είναι απαραίτητο για σύγχρονους προγραμματιστές

Με βάση τα σχόλια της κοινότητας Microsoft Foundry Discord, **το 45% των προγραμματιστών θέλει να χρησιμοποιήσει το AZD για εργασίες AI** αλλά αντιμετωπίζει προκλήσεις με:
- Πολύπλοκες αρχιτεκτονικές AI πολλαπλών υπηρεσιών
- Βέλτιστες πρακτικές ανάπτυξης AI σε παραγωγή  
- Ενσωμάτωση και διαμόρφωση υπηρεσιών Azure AI
- Βελτιστοποίηση κόστους για εργασίες AI
- Επίλυση προβλημάτων ανάπτυξης ειδικών για AI

### Μαθησιακοί Στόχοι

Ολοκληρώνοντας αυτό το δομημένο μάθημα, θα:
- **Κατανοήσετε τα Θεμελιώδη του AZD**: Βασικές έννοιες, εγκατάσταση, και διαμόρφωση
- **Αναπτύξετε Εφαρμογές AI**: Χρησιμοποιώντας AZD με υπηρεσίες Microsoft Foundry
- **Υλοποιήσετε Υποδομή ως Κώδικα**: Διαχείριση πόρων Azure με πρότυπα Bicep
- **Επιλύσετε Προβλήματα Ανάπτυξης**: Επίλυση κοινών σφαλμάτων και αποσφαλμάτωση
- **Βελτιστοποιήσετε για Παραγωγή**: Ασφάλεια, κλιμάκωση, παρακολούθηση, και διαχείριση κόστους
- **Δημιουργήσετε Πολυ-Πρακτορικές Λύσεις**: Ανάπτυξη σύνθετων αρχιτεκτονικών AI

#### Πρακτικές Ασκήσεις

```bash
# Αναπτύξτε την πρώτη σας εφαρμογή τεχνητής νοημοσύνης
azd init --template azure-search-openai-demo
azd up

# Δοκιμάστε επιπλέον πρότυπα τεχνητής νοημοσύνης
azd init --template openai-chat-app-quickstart
azd init --template agent-openai-python-prompty
```

**💡 Αποτέλεσμα Κεφαλαίου**: Αναπτύξτε και διαμορφώστε μια εφαρμογή συνομιλίας AI με δυνατότητες RAG

**✅ Επιβεβαίωση Επιτυχίας:**
```bash
# Μετά το Κεφάλαιο 2, θα πρέπει να μπορείτε να:
azd init --template azure-search-openai-demo
azd up
# Δοκιμάσετε το περιβάλλον συνομιλίας AI
# Κάνετε ερωτήσεις και να λαμβάνετε απαντήσεις με τεχνητή νοημοσύνη με πηγές
# Επαληθεύσετε ότι λειτουργεί η ενσωμάτωση αναζήτησης
azd monitor  # Ελέγξετε ότι το Application Insights εμφανίζει τηλεμετρία
azd down --force --purge
```

**📊 Επένδυση Χρόνου:** 1-2 ώρες  
**📈 Επίπεδο Δεξιότητας Μετά:** Μπορεί να αναπτύσσει και να διαμορφώνει παραγωγικές εφαρμογές AI  
**💰 Ενημέρωση Κόστους:** Κατανόηση κόστους ανάπτυξης $80-150/μήνα, κόστους παραγωγής $300-3500/μήνα

#### 💰 Εξετάσεις Κόστους για Αναπτύξεις AI

**Περιβάλλον Ανάπτυξης (Εκτιμώμενο $80-150/μήνα):**
- Azure OpenAI (πληρωμή ανά χρήση): $0-50/μήνα (βάσει χρήσης token)
- AI Search (βασική βαθμίδα): $75/μήνα
- Container Apps (κατανάλωση): $0-20/μήνα
- Αποθήκευση (Τυπική): $1-5/μήνα

**Περιβάλλον Παραγωγής (Εκτιμώμενο $300-3,500+/μήνα):**
- Azure OpenAI (PTU για συνεπή απόδοση): $3,000+/μήνα Ή Πληρωμή ανά χρήση με υψηλό όγκο
- AI Search (τυπική βαθμίδα): $250/μήνα
- Container Apps (αφιερωμένο): $50-100/μήνα
- Application Insights: $5-50/μήνα
- Αποθήκευση (Premium): $10-50/μήνα

**💡 Συμβουλές Βελτιστοποίησης Κόστους:**
- Χρησιμοποιήστε **Δωρεάν Επίπεδο** Azure OpenAI για εκμάθηση (περιλαμβάνει 50.000 tokens/μήνα)
- Εκτελέστε `azd down` για απελευθέρωση πόρων όταν δεν αναπτύσσετε ενεργά
- Ξεκινήστε με χρέωση βάσει κατανάλωσης, αναβαθμίστε σε PTU μόνο για παραγωγή
- Χρησιμοποιήστε `azd provision --preview` για εκτίμηση κόστους πριν την ανάπτυξη
- Ενεργοποιήστε αυτόματη κλιμάκωση: πληρώνετε μόνο για την πραγματική χρήση

**Παρακολούθηση κόστους:**
```bash
# Ελέγξτε τα εκτιμώμενα μηνιαία κόστη
azd provision --preview

# Παρακολουθήστε τα πραγματικά κόστη στο Azure Portal
az consumption budget list --resource-group <your-rg>
```

## 📖 Τι είναι το Azure Developer CLI;

Το Azure Developer CLI (azd) είναι μια γραμμή εντολών εστιασμένη στον προγραμματιστή που επιταχύνει τη διαδικασία δημιουργίας και ανάπτυξης εφαρμογών στο Azure. Παρέχει:

- **Αναπτύξεις με πρότυπα** - Χρήση προ-κατασκευασμένων προτύπων για κοινά πρότυπα εφαρμογών  
- **Υποδομή ως Κώδικας** - Διαχείριση πόρων Azure με Bicep ή Terraform  
- **Ενσωματωμένες ροές εργασίας** - Ομαλή παραχώρηση, ανάπτυξη και παρακολούθηση εφαρμογών  
- **Φιλικό στον προγραμματιστή** - Βελτιστοποιημένο για παραγωγικότητα και εμπειρία

### **AZD + Microsoft Foundry: Ιδανικό για Αναπτύξεις AI**

**Γιατί AZD για Λύσεις AI;** Το AZD αντιμετωπίζει τις κορυφαίες προκλήσεις που συναντούν οι προγραμματιστές AI:

- **Πρότυπα έτοιμα για AI** - Προ-διαμορφωμένα πρότυπα για Azure OpenAI, Cognitive Services και ML workloads  
- **Ασφαλείς Αναπτύξεις AI** - Ενσωματωμένα πρότυπα ασφάλειας για υπηρεσίες AI, κλειδιά API και endpoints μοντέλων  
- **Παραγωγικά Πρότυπα AI** - Καλές πρακτικές για κλιμακούμενες, αποδοτικές αναπτύξεις εφαρμογών AI  
- **Ολοκληρωμένες Ροές Εργασίας AI** - Από την ανάπτυξη μοντέλων μέχρι την παραγωγική ανάπτυξη με σωστή παρακολούθηση  
- **Βελτιστοποίηση Κόστους** - Έξυπνος καταμερισμός πόρων και στρατηγικές κλιμάκωσης workloads AI  
- **Ενσωμάτωση Microsoft Foundry** - Απρόσκοπτη σύνδεση σε κατάλογο μοντέλων και endpoints Microsoft Foundry

```bash
# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# Επαληθεύστε την εγκατάσταση
azd version
```

### ❌ "Δεν βρέθηκε συνδρομή" ή "Η συνδρομή δεν ορίστηκε"

```bash
# Λίστα διαθέσιμων συνδρομών
az account list --output table

# Ορισμός προεπιλεγμένης συνδρομής
az account set --subscription "<subscription-id-or-name>"

# Ορισμός για περιβάλλον AZD
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# Επιβεβαίωση
az account show
```

### ❌ "Ανεπαρκές Πόσο" ή "Υπέρβαση ποσόστωσης"

```bash
# Δοκιμάστε διαφορετική περιοχή Azure
azd env set AZURE_LOCATION "westus2"
azd up

# Ή χρησιμοποιήστε μικρότερα SKU στην ανάπτυξη
# Επεξεργαστείτε το infra/main.parameters.json:
{
  "sku": "B1"  // Instead of "P1V2"
}
```

### ❌ Αποτυχία "azd up" στη μέση της διαδικασίας

```bash
# Επιλογή 1: Καθαρίστε και ξαναδοκιμάστε
azd down --force --purge
azd up

# Επιλογή 2: Απλά διορθώστε την υποδομή
azd provision

# Επιλογή 3: Ελέγξτε λεπτομερή αρχεία καταγραφής
azd show
azd logs
```

### ❌ "Αποτυχία πιστοποίησης" ή "Η Λήξη του Token"

```bash
# Επαναπιστοποίηση
az logout
az login

azd auth logout
azd auth login

# Επαληθεύστε την αυθεντικοποίηση
az account show
```

### ❌ "Ο πόρος υπάρχει ήδη" ή συγκρούσεις ονομάτων

```bash
# Το AZD δημιουργεί μοναδικά ονόματα, αλλά σε περίπτωση σύγκρουσης:
azd down --force --purge

# Στη συνέχεια, επαναδοκιμάστε με νέο περιβάλλον
azd env new dev-v2
azd up
```

### ❌ Η ανάπτυξη προτύπου καθυστερεί πολύ

**Κανονικοί χρόνοι αναμονής:**
- Απλή web εφαρμογή: 5-10 λεπτά
- Εφαρμογή με βάση δεδομένων: 10-15 λεπτά
- Εφαρμογές AI: 15-25 λεπτά (η προετοιμασία OpenAI είναι αργή)

```bash
# Έλεγχος προόδου
azd show

# Εάν κολλήσει >30 λεπτά, ελέγξτε το Azure Portal:
azd monitor
# Αναζητήστε αποτυχημένες υλοποιήσεις
```

### ❌ "Άρνηση άδειας" ή "Απαγορευμένο"

```bash
# Ελέγξτε το ρόλο σας στο Azure
az role assignment list --assignee $(az account show --query user.name -o tsv)

# Χρειάζεστε τουλάχιστον το ρόλο "Συνεισφέροντα"
# Ζητήστε από τον διαχειριστή Azure να χορηγήσει:
# - Συνεισφέροντα (για πόρους)
# - Διαχειριστή Πρόσβασης Χρηστών (για αναθέσεις ρόλων)
```

### ❌ Δεν μπορεί να βρει το URL της αναπτυγμένης εφαρμογής

```bash
# Εμφάνιση όλων των σημείων τερματισμού υπηρεσιών
azd show

# Ή άνοιγμα του Azure Portal
azd monitor

# Έλεγχος συγκεκριμένης υπηρεσίας
azd env get-values
# Αναζήτηση μεταβλητών *_URL
```
