## သင်တန်း အနှစ်ချုပ်

Azure Developer CLI (azd) ကို ဆက်တိုက် သင်ကြားနိုင်သော အခန်းခွဲများဖြင့် အဆင့်လိုက် သင်ကြားနိုင်ပါသည်။ **Microsoft Foundry နှင့် ပေါင်းစပ်၍ AI application များ တင်ပို့ခြင်းအပေါ် အထူးအာရုံစိုက်မှု** ပါဝင်သည်။

### ဒီသင်တန်းကို နည်းပညာရှင်များအတွက် အရေးပါသည့် အကြောင်းရင်း

Microsoft Foundry Discord အသိုင်းအဝိုင်းမှ အမြင်အရ **ဖျော်ဖြေမှုလုပ်ငန်းအတွက် AZD ကို အသုံးပြုလိုသော developer ၄၅% ရှိသည်** သော်လည်း အခက်အခဲများနှင့် ကြုံတွေ့နေသည် -
- ရှုပ်ထွေးသော multi-service AI ဖွဲ့စည်းမှုများ
- ထုတ်လုပ်မှုအဆင့် AI တင်ပို့မှု လမ်းညွှန်ချက်များ
- Azure AI ၀န်ဆောင်မှု ပေါင်းစည်းခြင်းနှင့် ပြင်ဆင်မှု
- AI ဖျော်ဖြေမှုများအတွက် ကုန်ကျစရိတ် သက်သာစေရန်
- AI-specific deployment ပြဿနာများ ဖြေရှင်းခြင်း

### သင်ယူရမည့် ရည်ရွယ်ချက်များ

ဒီတစိတ်တပိုင်း သင်တန်းပြီးရင် သင်မှာ -
- **AZD အခြေခံများကျွမ်းကျင်မှု**: အဓိကအယူအဆများ၊ သွင်းယူခြင်းနှင့် ပြင်ဆင်မှု
- **AI Applications တင်ပို့ခြင်း**: AZD နဲ့ Microsoft Foundry ၀န်ဆောင်မှုများ အသုံးပြုခြင်း
- **Infrastructure as Code ကို အကောင်အထည်ဖော်ခြင်း**: Azure ကို Bicep templates ဖြင့် စီမံခန့်ခွဲခြင်း
- **Deployment ပြဿနာစီမံခန့်ခွဲခြင်း**: မဖြေရှင်းရသေးသော ပြဿနာများစနစ်တကျ ဖြေရှင်းခြင်း
- **ထုတ်လုပ်မှုအတွက် ထိရောက်စွာ၊ အပေါ်ပြုလုပ်ခြင်း**: လုံခြုံရေး၊ တိုးချဲ့မှု၊ စောင့်ကြည့်မှု၊ ကုန်ကျစရိတ် စီမံခန့်ခွဲမှု
- **Multi-Agent ဖြေရှင်းချက်များတည်ဆောက်ခြင်း**: ရှုပ်ထွေးသော AI ဖွဲ့စည်းမှုများ တင်ပို့ခြင်း

**အချိန်ယူဆောင်ရွက်မှု**: ၃၀-၄၅ မိနစ်  
**ရှုပ်ထွေးမှုအဆင့်**: ⭐

#### သင်ယူမည့်အရာများ
- Azure Developer CLI အခြေခံများကို နားလည်ခြင်း
- သင့်ပလက်ဖောင်းပေါ် AZD ကို ထည့်သွင်းခြင်း
- ပထမဆုံး အောင်မြင်သော deployment ပြုလုပ်ခြင်း

#### သင်ယူရေး အရင်းအမြစ်များ
- **🎯 စတင်ရန်**: [Azure Developer CLI ဆိုသည်မှာ?](../..)
- **📖 သီအိုရီ**: [AZD အခြေခံများ](docs/getting-started/azd-basics.md) - အဓိက အယူအဆနှင့် အသုံးအနှုန်းများ
- **⚙️ ပြင်ဆင်မှု**: [ထည့်သွင်းခြင်းနှင့် ပြင်ဆင်မှု](docs/getting-started/installation.md) - ပလက်ဖောင်းအလိုက် လမ်းညွှန်ချက်များ
- **🛠️ လက်တွေ့လုပ်ဆောင်မှု**: [ပထမဆုံးပရောဂျက်](docs/getting-started/first-project.md) - အဆင့်လိုက် လမ်းညွှန်ချက်
- **📋 အမြန်ညွှန်ကြားချက်**: [Command Cheat Sheet](resources/cheat-sheet.md)

#### လက်တွေ့ လေ့ကျင့်မှုများ
```bash
# အမြန်ဖွင့်ခြင်းစစ်ဆေးမှု
azd version

# သင့်ရဲ့ပထမဆုံးလျှပ်စစ် application ကို စတင်တင်သွင်းပါ
azd init --template todo-nodejs-mongo
azd up
```

**💡 အခန်းရလဒ်**: AZD ကို အသုံးပြု၍ ရိုးရှင်းသော ဝက်ဘ်အက်ပ်ကို Azure သို့ အောင်မြင်စွာ တင်ပို့ပါ

**✅ အောင်မြင်မှုအတည်ပြုချက်:**
```bash
# ပထမပိုင်း ကို စပြီး ပြီးစီးသွားဖို့နောက် ပြင်ဆင်နိုင်မယ်ဆိုရင်
azd version              # တပ်ဆင်ထားတဲ့ ဗားရှင်းကို ပြပါသည်
azd init --template todo-nodejs-mongo  # ပရောဂျက်ကို စတင်တည်ဆောက်သည်
azd up                  # Azure တွင် အပ်လုတ်ပြန်လုပ်သည်
azd show                # ပြေးနေသော အက်ပ်၏ URL ကို ပြပါသည်
# အက်ပ်သည် ဘရောက်ဇာတွင်ဖွင့်ပြီး လည်ပတ်သည်
azd down --force --purge  # အရင်းအမြစ်များကို ရှင်းလင်းသည်
```

**📊 ကုန်ချိန်:** ၃၀-၄၅ မိနစ်  
**📈 လေ့ကျင့်ပြီးနောက် ကျွမ်းကျင်မှု:** အခြေခံအက်ပ်များကို ခွဲထုတ်တင်ပို့နိုင်ခြင်း

**✅ အောင်မြင်မှုအတည်ပြုချက်:**
```bash
# အခန်း ၁ ကို ပြီးစီးပြီးနောက်၊ သင်သည် နားလည်စေရန် ဖြစ်သည်။
azd version              # တပ်ဆင်ထားသော ဗားရှင်းကို ပြသသည်
azd init --template todo-nodejs-mongo  # စီမံကိန်းကို အစပြုသည်
azd up                  # Azure သို့ ဖြန့်ဖြူးသည်
azd show                # လုပ်ဆောင်နေသော app URL ကို ပြသသည်
# အသုံးပြုမှု browser တွင် ဖွင့်ပြီး အလုပ်လုပ်သည်
azd down --force --purge  # အရင်းအမြစ်များကို သန့်ရှင်းသည်
```

**📊 ကုန်ချိန်:** ၃၀-၄၅ မိနစ်  
**📈 လေ့ကျင့်ပြီးနောက် ကျွမ်းကျင်မှု:** အခြေခံအက်ပ်များကို ခွဲထုတ်တင်ပို့နိုင်ခြင်း

- auto-scaling ကို ဖွင့်ထား၍ သုံးစွဲမှုအရသာပေးငွေနေစေရန်

## 📖 Azure Developer CLI ဆိုတာဘာလဲ?

Azure Developer CLI (azd) သည် Azure အတွက် အက်ပလီကေးရှင်း ဖန်တီးခြင်းနှင့် တပ်ဆင်ခြင်း လုပ်ငန်းစဉ်ကို မြန်ဆန်စေသော developer ကို အလွယ်တကူအသုံးပြုနိုင်သော command-line ရိုးရှင်းကိရိယာ ဖြစ်သည်။ ၎င်း သည်-

- **နမူနာအပေါ်ပြု၍ တပ်ဆင်ခြင်း** - များပြားသော app ပုံစံများအတွက် ကြိုတင်ပြင်ဆင်ထားသည့် နမူနာများကို အသုံးပြုနိုင်ခြင်း  
- **Infrastructure as Code** - Bicep သို့မဟုတ် Terraform အသုံးပြု၍ Azure အရင်းအမြစ်များ စီမံနိုင်ခြင်း  
- **ပေါင်းစပ် လုပ်ဆောင်မှုများ** - အက်ပလီကေးရှင်း များကို စုပေါင်း တပ်ဆင်၊ စစ်ဆေး နှင့် ကြည့်ရှုနိုင်ခြင်း  
- **Developer Friendly** - developer အတွက် အရည်အသွေးမြင့် ဖန်တီးမှုနှင့် အသုံးပြုမှုအတွေ့အကြုံများ

**ဘာဖြစ်လို့ AZD သည် AI ဖြေရှင်းချက်များအတွက် သင့်တော်သနည်း?** AI ဖန်တီးသူများကြုံတွေ့ရသော အကြီးစား စိန်ခေါ်မှုများကို AZD က ဖြေရှင်းပေးသည်-

- **AI-သင့် Template များ** - Azure OpenAI, Cognitive Services, ML အလုပ်များအတွက် ကြိုတင်ပြင်ဆင်ထားသည့် template များ  
- **လုံခြုံရေးရှိသော AI တပ်ဆင်မှုများ** - AI ဝန်ဆောင်မှုများ၊ API key များ၊ မော်ဒယ် endpoint များအတွက် လုံခြုံရေးပုံစံများ  
- **ထုတ်လုပ်မှုပေးသော AI ပုံစံများ** - ချဲ့ထွင်နိုင်ပြီး စရိတ်ကျမည့် AI app များအတွက် အကောင်းဆုံး လမ်းညွှန်ချက်များ  
- **အစမှ အဆုံး AI လုပ်ငန်းစဉ်များ** - မော်ဒယ် ဖန်တီးမှ ထုတ်လုပ်မှုအထိ စနစ်တကျ ဆောင်ရွက်နိုင်ခြင်း  
- **ကုန်ကျစရိတ် ထိရောက်စွာ စီမံခြင်း** - အရင်းအမြစ် များကို ဉာဏ်ရည်စွမ်းသော စီမံခန့်ခွဲမှုနှင့် ချဲ့ထွင်မှု  
- **Microsoft Foundry ချိတ်ဆက်မှု** - Microsoft Foundry မော်ဒယ်များ၏ ကတ်တရာနှင့် endpoint များ သွားရာ အလျင်အမြန် ချိတ်ဆက်ခြင်း

## 🔧 အမြန်ပြဿနာဖြေရှင်းလမ်းညွှန်

**အစပြုသူများ၏ ပုံမှန် ကြုံတွေ့ရသော ပြဿနာများနှင့် ချက်ချင်း ဖြေရှင်းနည်းများ။**

### ❌ "azd: command not found"

```bash
# အရင်ဆုံး AZD ကိုတပ်ဆင်ပါ
# Windows (PowerShell):
winget install microsoft.azd

# macOS:
brew tap azure/azd && brew install azd

# Linux:
curl -fsSL https://aka.ms/install-azd.sh | bash

# တပ်ဆင်မှုကိုစစ်ဆေးပါ
azd version
```

### ❌ "No subscription found" or "Subscription not set"

```bash
# အသုံးပြုနိုင်သော စာရင်းသွင်းမှုများကို ပြပါ
az account list --output table

# ပုံမှန်စာရင်းသွင်းမှု သတ်မှတ်ပါ
az account set --subscription "<subscription-id-or-name>"

# AZD ပတ်ဝန်းကျင်အတွက် သတ်မှတ်ပါ
azd env set AZURE_SUBSCRIPTION_ID "<subscription-id>"

# အတည်ပြုပါ
az account show
```

### ❌ "InsufficientQuota" or "Quota exceeded"

```bash
# ခြောက်လှမ်းသော Azure ဒေသကို စမ်းသပ်ပါ
azd env set AZURE_LOCATION "westus2"
azd up

# သို့မဟုတ် ဖွံ့ဖြိုးရေးတွင် ရှယ်ယာသေးငယ်များကို အသုံးပြုပါ
# infra/main.parameters.json ကို တည်းဖြတ်ပါ:
{
  "sku": "B1"  // Instead of "P1V2"
}
```

### ❌ "azd up" fails halfway through

```bash
# ရွေးချယ်မှု ၁: သန့်ရှင်းပြီး ထပ်မံကြိုးစားပါ
azd down --force --purge
azd up

# ရွေးချယ်မှု ၂: အခြေခံအဆောက်အအုံသာ ပြင်ဆင်ပါ
azd provision

# ရွေးချယ်မှု ၃: အသေးစိတ် မှတ်တမ်းများကို စစ်ဆေးပါ
azd show
azd logs
```

### ❌ "Authentication failed" or "Token expired"

```bash
# ထပ်မံအတည်ပြုခြင်း
az logout
az login

azd auth logout
azd auth login

# အတည်ပြုမှုကိုစစ်ဆေးပါ
az account show
```

### ❌ "Resource already exists" or naming conflicts

```bash
# AZD သည် ထူးခြားသော နာမည်များ ထုတ်လုပ်ပေးသော်လည်း၊ မှားယွင်းမှု ရှိပါက
azd down --force --purge

# ထိုအခါ အသစ်သော ပတ်ဝန်းကျင်ဖြင့် ထပ်မံကြိုးစားပါ။
azd env new dev-v2
azd up
```
